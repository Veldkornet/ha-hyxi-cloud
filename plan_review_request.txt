The CI failure is caused by `mypy` complaining about `val` being passed to `float()` when its type is inferred as `Any | None`. In `_get_metric_float`:
```python
    def _get_metric_float(self, key: str) -> float | None:
        """Safely extract a metric value as a float."""
        val = self._metrics.get(key)

        if is_null_value(val):
            return None

        try:
            return float(val)
```
Although `is_null_value(val)` returns `True` for `None`, mypy does not know that `is_null_value` acts as a TypeGuard narrowing `val` out of `None`. We can cast `val` to `float(val) # type: ignore` or assert `val is not None` before `return float(val)`. Another option is just checking `if val is None or is_null_value(val): return None`.
